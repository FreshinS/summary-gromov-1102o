Основы GitHub

Ч.1 Основные понятия GitHub

GitHub - это платформа хостинга, хранения и совместной разработке. Здесь разработчики могут хранить свои проекты, работать над ними в команде, самостоятельно, параллельно и отслеживать изменения, обновления.

Для использования GitHub мы ис пользуем репозитории - места, где хранится код проекта, вместе со всей его историей изменений.

Чтобы получить доступ к репозиторию нужно иметь SSH-ключ
SSH-ключ - это способ безопасной аутентификации (подключение) на удаленном (облачном) сервере. Он используется для подключения к GitHub, например, без постоянного ввода пароля.

Коммит - это сохранение изменений в коде проекта. Он создает точку в истории, которую можно сохранить и вернуться к ней. Благодаря коммитам сразу несколько людей могут работать над одним проектом, и не превращать его в кашу

Коммиты обычно вводятся в ветки - отдельные линии разработки, которые могут быть созданы для работы над новой версией (функцией) или исправлением, не затрагивая основной код проекта.

Ч.2 Основные инструкции к GitHub

Ч.2.1 Как сгенерировать SSH-ключ и добавить к себе на GitHub

  1. Откройте терминал и выполните 
    ssh-keygen -t rsa -b 4096
// ssh-keygen: Команда для создания нового ключа SSH.
// -t rsa: Указывает тип ключа, который следует создать.
// -b 4096: Задает размер бит ключа.
  2. Нажмите Enter, чтобы принять стандартное расположение файла.
  3. Введите пароль или нажмите Enter, чтобы оставить его пустым.
  4. Выполните, чтобы скопировать (ПКМ) открытый ключ. 
    cat ./.ssh/id_rsa.pub
// cat: выводит содержимое файла в консоль
// './.ssh/id_rsa.pub': путь до файла с ключом
  5. Перейдите в настройки GitHub -> SSH and GPG keys -> New SSH key -> Вставьте ключ.

Ч.2.2 Как добавить к себе чей-то репозиторий

  1. Скопируйте URL репозитория с GitHub (зеленая кнопка Code).
  2. В терминале выполните 
    git clone <URL репозитория>

Ч.2.3 Как добавить на свой GitHub склонированный репозиторий

  1. Создайте пустой репозиторий на GitHub.
  2. Скопируйте его SSH-ссылку.
  3. Войдите в директорию склонированного репозитория в терминале.
  4. Выполните 
    git remote set-url origin <SSH-link>
// git remote: Команда для взаимодействия с удалёнными репозиториями
// set-url: Позволяет установить новый URL-адрес для удаленного репозитория.
// origin: Имя удаленного репозитория, обычно это origin
// <SSH-link>: SSH-ссылка на ваш репозиторий

Ч.2.4 Как сохранить изменения на GitHub (сделать коммит)

  1. Выполните, чтобы добавить все изменения.
    2. git add -A
// или
git add -all
  2. Выполните
    3. git commit -m "Описание коммита"
  3. Выполните, чтобы отправить изменения на GitHub.
    4. git push

Ч.2.5 Как работать со своим репозиторием с другого устройства

  1. Склонируйте свой репозиторий с GitHub на другое устройство
    2. git clone <URL вашего репозитория>
  2. Вносите изменения, коммитите и отправляйте их обратно на GitHub.

Ч.2.6 Как открыть VSCode из терминала Linux:

  1. В терминале выполните для открытия текущего каталога в VSCode.
    2. code <имя-файла(папки)>

Ч.2.7 Как запустить программу в терминале (через Node.js)

    node имя_файла.js

Ч.2.8 Как создать ветку

  1. Выполните, чтобы создать новую ветку.
    2. git branch <Новая ветка>
  2. Выполните, чтобы переключиться на новую ветку.
    3. git checkout <Новая ветка>
  3. *Или используйте для создания и переключения на новую ветку сразу.
    4. git checkout -b <Новая ветка>

Ч.3 Часто задаваемые вопросы

  1. В каком случае нужно генерировать новый SSH-ключ:
    • Когда вы впервые начинаете работу с GitHub.
    • При использовании нового компьютера или сервера для работы с репозиториями.
    • Когда старый ключ утерян или скомпрометирован.
    1.1 Могу ли я использовать один и тот же SSH-ключ на разных устройствах:
    • Да, вы можете использовать один и тот же SSH-ключ на разных устройствах. Это удобно при работе с несколькими устройствами.
  2. В каких случаях надо использовать git config user.name (user.email) <аргументы>:
    • При первой настройке Git на новом устройстве.
    • При изменении вашего имени или электронной почты.
    • Когда вам нужно использовать разные имена или адреса электронной почты для разных проектов.
  3. В каком случае нужно создавать ветку и для чего она нужна:
    • Новая функциональность: создайте ветку для работы над новой функцией, чтобы не затрагивать основной код.
    • Исправление ошибок: создайте ветку для исправления ошибок, чтобы избежать конфликтов с основной разработкой.
    • Разработка параллельных версий: создайте ветку для разработки и тестирования новых идей без влияния на стабильный код.
  4. Что писать в комментарии для git commit:
    • Краткое, но информативное описание изменений, чтобы понять суть коммита.
    • Укажите, какие задачи решает этот коммит.
    • Следуйте принципу "Как" и "Почему" для более полного объяснения.
  5. Что такое Pull Request и зачем он нужен:
    • Pull Request (или Merge Request) - это запрос на внесение изменений из вашей ветки в основную ветку проекта.
    • Используется для обсуждения и рецензии ваших изменений перед их объединением с основным кодом.
    • Позволяет совершать изменения согласованно и избегать конфликтов при слиянии кода.
